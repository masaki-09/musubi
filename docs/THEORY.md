# 関係性暗号 — 理論と思想

> *本書は musubi の設計思想と直感的な解説です。形式仕様は [SPEC.md](SPEC.md) を参照してください。*

## 1. なぜ「関係性」なのか

古典暗号の歴史を眺めると、ほぼすべての方式は二つの操作から成り立っています。

| 操作 | 例 | 直感 |
|---|---|---|
| **置換 (substitution)** | シーザー暗号、ヴィジュネル | 「`A` → `D`」のように、ある文字を別の文字に置き換える |
| **転置 (transposition)** | スキュタレー、レールフェンス  | 文字の **順序** を並び替える |

musubi の関係性暗号は、このどちらでもありません。

**文字そのものを書かない。** その代わりに「文字同士がどう関係しているか」だけを記述します。一つだけ「アンカー文字」を本物として明示し、残りはアンカーから順に「○番目の◯個後」「○番目と同じ」「○番目の鏡像」として連鎖的に表現します。

> **例**：平文「あいしてる」（アンカー = 3番目「し」）
>
> ```
> 1番目: 2番目から ―2 移動
> 2番目: 3番目（し）から ―3 移動
> 3番目: 「し」（アンカー）
> 4番目: 3番目（し）から +7 移動
> 5番目: 4番目から ―1 移動
> ```
>
> 鍵 (五十音の並び順) を知っていれば、この「関係の連鎖」から平文が一意に復元できます。

## 2. アンカーという発明

関係だけでは、絶対の意味は決まりません。「私の左隣の人は私より背が高い」という関係を100人分集めても、誰一人として身長は分かりません。**最低1人の身長を教えてもらえば**、そこから全員の身長が確定します。

musubi のアンカーはまさにこれです。アンカーを知る者と知らぬ者の間に、復号可能性という大きな崖が生まれます。

そしてアンカーは**位置と文字の組**として表現されるので、暗号文を読む者は最初に「ここが3番目で『し』である」という情報を受け取ります。これが詩的に良い：謎解きが「3番目の『し』」という確かな足場から始まるのです。

## 3. 鍵 = 並び順

musubi の鍵は **アルファベット Σ の順列** です。たとえば `default-v1` には 175 文字あり、これらをどんな順番に並べるかで `175! ≈ 5.4 × 10³¹⁹` 通りの鍵が定義できます。

「あ」が 0 番目で「い」が 1 番目、というのは公開された並びです。鍵では「☃ → A → ぱ → … → ぐ」のようにシャッフルされ、各文字に**ランク (順位)** が割り当てられます。シフト・ミラー・同一の関係はすべて、このランク上の演算です。

## 4. 三種類の関係 (v1)

| 関係 | 数式 | 詩的な言い換え |
|---|---|---|
| `same`     | `idx_i = idx_ref` | 「同じ」 |
| `shift(k)` | `idx_i = idx_ref + k mod \|Σ\|` | 「k 個ずらす」 |
| `mirror`   | `idx_i = -idx_ref mod \|Σ\|` | 「鏡映」 |

`shift` ひとつで `same` (= `shift(0)`) と `mirror` (位置依存のシフト) を表現することは可能ですが、あえて分けたのは**詩的出力**のためです。「同じ」「対」「鏡」という言葉は、無味乾燥な「0個ずらす」より美しく謎を含みます。

## 5. 古典暗号としての強さ

弱いです。

それは弱点ではなく、**特徴**です。musubi は「現代暗号と渡り合える新方式」を目指していません。目指しているのは：

- **理論的な新規性**: 置換でも転置でもない第三の方式
- **詩的な暗号文**: 出力が「謎かけ」として読める
- **手紙としての魅力**: 紙に手書きしても運用可能な簡潔さ

頻度分析・既知平文攻撃・反復文字検出など、教科書的な手法でいくらでも破れます。これは仕様であり、バグではありません。詳細は [SECURITY.md](../SECURITY.md) を参照してください。

## 6. 拡張のアイデア

将来的に検討している方向：

- **連鎖参照のバリエーション**：現在の v1 は「アンカーへ向かって 1 ステップずつ」だけ。スキップ参照や複数アンカーで、もっと複雑な「謎」を作れる。
- **新しい関係タイプ**：`row_shift`（五十音表の行/段移動）、`voicing`（清濁切替）、`rhyme`（同じ母音）など、文字の構造的な関係。
- **詩的レンダラ**：JSON 暗号文を「百人一首風」「古文書風」「童謡風」に書き換えて出力する。
- **複数アンカー**：「`3番目は『し』、7番目は『て』」のように複数の足場を与えると、別々の連鎖を編むことができる。理論的には鍵空間は変わらないが、暗号文の自由度が上がる。

## 7. 「ロマン」とは何か

これは個人的な思想です。

暗号は本来、**情報を守るための道具**です。しかし古典暗号には別の側面があります — 知性のゲーム、手紙の遊戯、二人だけの言語。シーザー暗号を「強い」と呼ぶ人はいませんが、それを否定する人もいません。子供の頃に作った「秘密の暗号」は、解かれることを期待していませんでした。**作るこ自体が目的**でした。

musubi はそういう暗号でありたい。手紙を出すたびに鍵を変え、解いた相手が「3番目の『し』」から始まる謎を辿って意味を組み立てる、その時間そのものに価値を置きたい。

それが、この「全く新しい理論で、けれど現代的には脆弱な」古典暗号を作った理由です。
