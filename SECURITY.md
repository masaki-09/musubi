# セキュリティポリシー / Security Policy

## ⚠ 重要：musubi は玩具暗号です

**musubi は古典暗号の楽しみのためのソフトウェアであり、本気の秘匿通信や情報保護を意図していません。**

- 暗号方式（関係性暗号）は古典的な手法であり、現代の暗号解析技術を持つ攻撃者にとっては容易に破られます。
- 鍵生成・鍵管理・乱数源は、本物の暗号ソフトウェアに要求される基準を満たしていません。
- **本ソフトウェアを実際の機密情報の保護に使わないでください。**

恋文・パズル・自作古典暗号の楽しみのためにご利用ください 🪢

## 脆弱性の報告

以下のような **実装上の問題** を発見した場合は、ぜひご報告ください：

- パニック・メモリ安全性問題（Rust の `unsafe` を使っていない以上、ほぼ起こらないはずですが）
- DoS（極端に大きな入力で OOM など）
- 仕様書 (`docs/SPEC.md`) と実装の不一致
- 鍵生成における重大な偏り

### 報告方法

1. **公開可能な実装バグ**: GitHub の通常の [Issue](https://github.com/masaki-09/musubi/issues) で報告してください。
2. **慎重な扱いを要する問題**: GitHub の [Private Vulnerability Reporting](https://github.com/masaki-09/musubi/security/advisories/new) からご報告ください。

### 暗号強度に関する報告は不要です

「鍵空間が小さい」「頻度分析で破れる」等は **既知かつ意図的な性質** です。これは仕様であり、バグではありません。

## 謝辞

問題を報告いただいた方には、希望に応じて CHANGELOG / Release notes でお名前を記載させていただきます。
